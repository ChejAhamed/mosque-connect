Select Component Fix - Version 80.2

Previous Issue:

Uncaught Error: A <Select.Item /> must have a value prop that is not an empty string. This is because the Select value can be set to an empty string to clear the selection and show the placeholder.

The error occurred when SelectItem components were initialized with empty string values, which is not allowed by the Radix UI Select component. This caused runtime errors when filtering businesses and events.

Fix Applied:

  • Changed empty string values to "all" for consistency
  • Enhanced SelectItem component with error prevention
  • Updated filter logic to handle "all" value correctly
  • Added defensive coding to prevent future issues

The fix applies to the business, events, and hadith pages, ensuring consistent and error-free filtering across the application.

This page serves as a deployment reference. Access the full application at your Netlify URL once deployment is complete.
